Penyeimbang Data (Data UCL Ecoli)#

1. Apa itu Penyeimbangan Data?#

Penyeimbangan data adalah suatu teknik dalam pengolahan data yang digunakan untuk menangani masalah ketidakseimbangan kelas (class imbalance) pada dataset. Ketidakseimbangan ini muncul ketika jumlah data dari satu kelas (mayoritas) jauh lebih banyak dibanding kelas lainnya (minoritas).

Ketika kondisi ini terjadi, algoritma machine learning cenderung lebih mudah mempelajari pola dari kelas mayoritas karena data yang tersedia lebih banyak. Sebaliknya, pola dari kelas minoritas kurang terwakili sehingga model berisiko bias dan membuat prediksi yang tidak akurat terhadap kelas tersebut.

2. Contoh Sederhana Ketidakseimbangan Data#

Misalnya pada dataset deteksi penipuan transaksi (fraud detection):

99% transaksi = normal

1% transaksi = fraud

Jika sebuah model hanya memprediksi semua transaksi sebagai normal, maka:

Akurasi model = 99%

Namun, transaksi fraud tidak pernah terdeteksi, padahal itu adalah informasi paling penting.

Contoh ini menunjukkan bahwa akurasi tinggi tidak selalu berarti model baik, terutama saat dataset tidak seimbang.

3. Mengapa Penyeimbangan Data Diperlukan?#

Penyeimbangan data sangat penting karena sebagian besar algoritma machine learning mengasumsikan bahwa distribusi kelas dalam dataset relatif sama. Tanpa penyeimbangan, konsekuensinya:

Pola minoritas terabaikan Model tidak dapat belajar dengan baik dari kelas minoritas karena jumlah datanya terlalu sedikit.

Prediksi tidak adil Hasil prediksi lebih condong ke kelas mayoritas, sementara kelas minoritas jarang diprediksi.

Tingkat kesalahan tinggi pada kelas minoritas Meskipun akurasi total terlihat tinggi, kesalahan klasifikasi pada kelas minoritas bisa sangat besar.

Dengan melakukan penyeimbangan data, manfaat yang diperoleh antara lain:

Model lebih fair terhadap semua kelas.

Pola dari kelas minoritas bisa lebih mudah dipelajari.

Bias prediksi dapat diminimalkan.

Meningkatkan metrik evaluasi lain seperti precision, recall, dan F1-score, terutama pada kelas minoritas.

4. Metode Penyeimbangan Data#

Ada beberapa cara yang umum digunakan untuk menyeimbangkan data, antara lain:

Oversampling

Menambahkan jumlah data pada kelas minoritas.

Bisa dilakukan dengan menduplikasi data minoritas atau menggunakan teknik seperti SMOTE (Synthetic Minority Oversampling Technique) untuk membuat data sintetis baru.

Undersampling

Mengurangi jumlah data pada kelas mayoritas sehingga lebih seimbang dengan kelas minoritas.

Kekurangannya, sebagian informasi dari data mayoritas bisa hilang.

Hybrid Method (Gabungan)

Kombinasi dari oversampling dan undersampling agar distribusi data lebih proporsional.

Metode berbasis algoritma

Menggunakan algoritma khusus yang lebih tangguh terhadap data tidak seimbang, misalnya Balanced Random Forest atau XGBoost dengan class weight.

Beberapa jenis sel bakteri memiliki banyak sampel,

Jika dataset tidak diseimbangkan, model yang dilatih bisa gagal mengenali pola dari jenis bakteri yang jarang muncul. Oleh karena itu, langkah penyeimbangan data menjadi kunci utama agar model dapat:

Mendeteksi semua jenis bakteri dengan lebih akurat.

Penyeimbangan Data Menggunakan ADASYN#

1. Apa itu ADASYN?

ADASYN (Adaptive Synthetic Sampling Approach for Imbalanced Learning) adalah salah satu metode oversampling yang digunakan untuk menangani masalah class imbalance. Metode ini merupakan pengembangan dari SMOTE (Synthetic Minority Oversampling Technique).

Kalau pada SMOTE data sintetis ditambahkan secara merata pada seluruh sampel kelas minoritas, ADASYN menggunakan pendekatan yang lebih adaptif. Artinya, jumlah data sintetis yang dibuat untuk setiap sampel minoritas akan menyesuaikan dengan tingkat kesulitan sampel tersebut untuk dipelajari oleh model.

2. Cara Kerja ADASYN

Konsep utama ADASYN adalah belajar lebih banyak dari kasus minoritas yang sulit diklasifikasikan.

Sampel minoritas yang berada di sekitar banyak sampel mayoritas → dianggap sulit dipelajari. → ADASYN akan membuat lebih banyak data sintetis pada area ini.

Sampel minoritas yang lingkungannya relatif aman (lebih sedikit mayoritas di sekitarnya) → dianggap lebih mudah dipelajari. → ADASYN akan membuat lebih sedikit data sintetis di area ini.

Dengan strategi ini, ADASYN tidak hanya menyeimbangkan jumlah data, tapi juga membantu model fokus belajar pada bagian data yang paling menantang.

3. Perbedaan SMOTE dan ADASYN Aspek SMOTE ADASYN Distribusi data sintetis Dibuat merata untuk semua sampel minoritas Disesuaikan berdasarkan kesulitan sampel Fokus Menambah data minoritas secara umum Menambah data terutama pada area yang sulit dipelajari Tujuan Mengurangi ketidakseimbangan kelas Mengurangi ketidakseimbangan + membantu model lebih adaptif

4. Mengapa ADASYN Bermanfaat?

Lebih cerdas dari SMOTE: karena tidak menambah data secara sembarangan, melainkan hanya pada titik-titik yang memang “butuh perhatian lebih”.

Meningkatkan kinerja model: terutama dalam mendeteksi kelas minoritas yang sulit dibedakan dari mayoritas.

Mengurangi bias prediksi: model tidak hanya fokus pada mayoritas, tapi juga lebih peka terhadap minoritas yang sebelumnya terabaikan.

Membantu generalisasi: dengan data sintetis yang variatif pada area sulit, model bisa belajar representasi yang lebih baik.

Penyeimbangan Data dengan SMOTE#

1. Apa itu SMOTE?

SMOTE (Synthetic Minority Over-sampling Technique) adalah metode yang digunakan untuk mengatasi ketidakseimbangan kelas (class imbalance) pada dataset di pembelajaran mesin.

Berbeda dengan metode sederhana yang hanya menggandakan sampel minoritas, SMOTE membuat sampel sintetis baru dengan cara menginterpolasi antar sampel minoritas yang sudah ada. Dengan demikian, dataset menjadi lebih seimbang, namun tetap mempertahankan variasi alami data.

2. Cara Kerja SMOTE

Prinsip dasar SMOTE adalah menciptakan data minoritas tambahan secara cerdas, bukan sekadar menyalin data lama. Langkah-langkahnya:

Pilih satu sampel minoritas dari dataset.

Cari tetangga terdekat (k-nearest neighbors) dari sampel minoritas tersebut. Hanya tetangga yang juga merupakan minoritas yang dipertimbangkan.

Pilih satu tetangga secara acak dari hasil pencarian.

Buat titik baru di antara sampel dan tetangganya dengan interpolasi linier:

Titik baru = sampel + random(0,1) × (tetangga − sampel)

Proses ini diulang untuk semua sampel minoritas sampai jumlah data minoritas cukup seimbang dengan kelas mayoritas.

3. Kelebihan SMOTE

Mengurangi risiko overfitting dibanding metode duplikasi sederhana, karena data baru bukan salinan, melainkan interpolasi.

Memperluas representasi minoritas, sehingga model dapat belajar pola minoritas dengan lebih baik.

Meningkatkan kinerja model pada kelas minoritas yang sebelumnya jarang muncul.

4. Contoh Ilustrasi

Misalkan ada tiga sampel minoritas di ruang 2D:

Titik A, B, dan C.

SMOTE bisa membuat titik baru di antara A dan B, atau B dan C, sehingga jumlah data minoritas bertambah.

Titik baru ini berada di area “antar-titik” minoritas, bukan di area mayoritas, sehingga model bisa belajar pola minoritas dengan lebih efektif.

from sklearn.model_selection import train_test_split
from sklearn.naive_bayes import GaussianNB
from sklearn.metrics import classification_report, confusion_matrix, ConfusionMatrixDisplay

#2. Fungsi Evaluasi

def evaluasi_model(X, y, title=""):
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42, stratify=y)
    model = GaussianNB()
    model.fit(X_train, y_train)
    y_pred = model.predict(X_test)

    print(f"=== Hasil Evaluasi Naive Bayes {title} ===")
    print(classification_report(y_test, y_pred, zero_division=0))

    # Plot confusion matrix
    disp = ConfusionMatrixDisplay(confusion_matrix(y_test, y_pred), display_labels=model.classes_)
    disp.plot(cmap="Blues", xticks_rotation=45)
    plt.title(f"Confusion Matrix {title}")
    plt.show()
