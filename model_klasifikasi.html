Model Klasifikasi pada Dataset Ecoli#

1. Definisi#

Klasifikasi adalah suatu metode dalam pembelajaran mesin (machine learning) yang digunakan untuk mengelompokkan data ke dalam kelas tertentu berdasarkan sekumpulan fitur (atribut) yang dimilikinya. Metode ini termasuk dalam kategori supervised learning, karena model dibangun dan dilatih menggunakan data yang sudah dilabeli (fitur + kelas/target). Dengan kata lain, model “belajar” dari data historis untuk dapat mengenali pola dan kemudian mengaplikasikan pengetahuan tersebut pada data baru.

2. Tujuan#

Penerapan metode klasifikasi memiliki beberapa tujuan utama, antara lain:

  • Memprediksi kelas dari data baru: Model mampu mengklasifikasikan sampel yang belum pernah dilihat sebelumnya.

  • Menggali pola hubungan: Menemukan keterkaitan antara variabel input (fitur) dengan label kelas yang menjadi target.

  • Meningkatkan pengambilan keputusan: Memberikan informasi berbasis data yang membantu dalam analisis maupun aplikasi nyata (misalnya diagnosis medis, deteksi anomali, dan lain-lain).

3. Tahapan Utama dalam Proses Klasifikasi#

  1. Pengumpulan Data

Dataset yang digunakan harus sudah berlabel, artinya setiap baris data memiliki nilai atribut (fitur) dan informasi kelas (target).

  1. Pra-pemrosesan Data

  • Membersihkan data (menghapus nilai kosong/duplikat).

  • Melakukan normalisasi atau standarisasi jika dibutuhkan.

  • Menangani masalah ketidakseimbangan kelas (class imbalance), misalnya dengan teknik oversampling, undersampling, atau SMOTE.

  1. Pembagian Dataset Dataset biasanya dibagi menjadi data training dan testing, misalnya dengan rasio 80:20. Pada kasus klasifikasi, pembagian data sering menggunakan teknik stratified split, agar distribusi kelas tetap proporsional.

  2. Pelatihan Model Model klasifikasi dibangun dengan melatih algoritma tertentu menggunakan data training. Pada tahap ini, model “belajar” pola hubungan antara fitur dengan label.

  3. Evaluasi Model Model yang telah dilatih kemudian diuji menggunakan data testing untuk mengukur kinerjanya. Berbagai metrik evaluasi digunakan, seperti akurasi, precision, recall, F1-score, hingga confusion matrix.

  4. Prediksi Data Baru Setelah model dianggap cukup baik, ia dapat digunakan untuk memprediksi kelas dari data baru yang belum memiliki label.

4. Algoritma yang Digunakan#

a. Naive Bayes (NB)

  • Berdasarkan Teorema Bayes dalam probabilitas.

  • Mengasumsikan bahwa setiap fitur bersifat independen (tidak saling memengaruhi).

  • Sederhana, cepat, serta efektif untuk dataset teks (misalnya klasifikasi email spam vs non-spam).

b. Random Forest

  • Termasuk metode ensemble learning yang menggabungkan banyak Decision Tree.

  • Setiap pohon keputusan dilatih menggunakan subset data dan subset fitur secara acak (bootstrap sampling).

  • Prediksi akhir diperoleh dengan voting mayoritas dari semua pohon.

  • Lebih stabil, lebih tahan terhadap overfitting, serta akurat dibanding hanya menggunakan satu Decision Tree.

c. Bagging Naive Bayes

  • Menggunakan teknik Bootstrap Aggregating (Bagging) untuk membuat beberapa model Naive Bayes pada subset data yang diacak.

  • Setiap model memberikan prediksi, kemudian digabungkan dengan voting untuk menghasilkan keputusan akhir.

  • Tujuannya adalah mengurangi variansi pada Naive Bayes dan meningkatkan performa pada dataset tertentu.

d. Bagging Decision Tree

  • Sama seperti Bagging Naive Bayes, tetapi model dasar yang digunakan adalah Decision Tree.

  • Menghasilkan banyak Decision Tree yang dilatih pada subset data acak.

  • Mirip dengan Random Forest, namun Random Forest biasanya juga melakukan randomisasi fitur di setiap node, sedangkan Bagging fokus pada subset data.

5. Evaluasi Kinerja Model Klasifikasi#

Untuk menilai seberapa baik model dalam melakukan prediksi, digunakan beberapa ukuran kinerja berikut:

  • Akurasi: Persentase jumlah prediksi yang benar dibandingkan seluruh data uji.

  • Precision: Tingkat ketepatan prediksi positif, yaitu seberapa banyak data yang diprediksi positif benar-benar positif.

  • Recall (Sensitivity): Kemampuan model dalam menemukan semua data yang seharusnya positif.

  • F1-score: Rata-rata harmonis antara precision dan recall, digunakan ketika terdapat ketidakseimbangan kelas.

  • Confusion Matrix: Matriks yang menggambarkan distribusi prediksi model dibandingkan label sebenarnya (True Positive, True Negative, False Positive, False Negative).
