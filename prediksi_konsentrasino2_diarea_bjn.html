Prediksi Konsentrasi NO₂ di Kabupaten Bojonegoro dengan KNN Regression#

Tahap Pengumpulan Data Penjelasan#

Data yang digunakan adalah konsentrasi gas Nitrogen Dioksida (NO₂) hasil observasi satelit Sentinel-5P yang disediakan melalui platform Copernicus OpenEO.

Rentang waktu pengambilan data: 1 Juni 2025 – 19 Oktober 2025, menghasilkan data harian rata-rata konsentrasi NO₂.

Data diambil untuk wilayah Kabupaten Bojonegoro, dengan batas koordinat poligon berikut:

Data NO₂ diambil dari produk Sentinel-5P Level-2 (S5P L2) melalui platform Copernicus/OpenEO. Sentinel-5P membawa instrumen TROPOMI yang mengukur kolom gas atmosfer (mis. NO₂, O₃). Produk L2 menyajikan pengukuran pada grid spasial yang relatif halus untuk skala regional. Saat mengambil data melalui OpenEO kita perlu:

Memilih koleksi yang tepat: pastikan SENTINEL_5P_L2 benar-benar berisi band NO2 (atau nama band sesuai documentasi layanan). Produk L2 biasanya mengandung metadata quality (mis. QA_value) yang bisa dipakai untuk filter pengukuran berkepercayaan rendah.

Spasial AOI: gunakan polygon yang mewakili batas wilayah (di sini kotak di Bojonegoro). Penting memeriksa apakah AOI terlalu kecil sehingga jumlah piksel valid sedikit, yang dapat meningkatkan noise.

Temporal extent: tentukan rentang tanggal (1 Juni — 19 Okt 2025). Frekuensi TROPOMI harian, tapi ada kasus data cloud/qa mengurangi observasi valid di beberapa hari — oleh karena itu agregasi harian dan pengecekan missing penting.

Masking & quality filtering: beberapa piksel memiliki nilai negatif atau flagged sebagai invalid (mis. cloud cover, low QA). Langkah mask untuk menghapus nilai < 0 perlu digabungkan jika tersedia filter QA_value dari produk agar tidak menggunakan data beresiko.

Agregasi: kita mengambil rata-rata temporal per hari lalu rata-rata spasial di dalam AOI. Ini menghasilkan seri waktu harian (satu nilai NO₂ per hari) yang lebih mudah diolah ketimbang citra penuh.

Catatan operasional: saat menjalankan batch job di OpenEO, perhatikan kuota, ukuran job, dan nama output — serta cek log job untuk error/timeout.

# ==========================
# Tahap 1: Pengumpulan Data
# ==========================
import openeo
import pandas as pd
import matplotlib.pyplot as plt
import warnings
warnings.filterwarnings("ignore")

# Autentikasi ke platform OpenEO Copernicus
connection = openeo.connect("openeo.dataspace.copernicus.eu").authenticate_oidc()

# Area of Interest (AOI)
aoi_geojson = {
    "type": "Polygon",
    "coordinates": [[
        [111.84013944486253, -7.112176997696622],
        [111.84013944486253, -7.201964283327371],
        [111.93244545164555, -7.201964283327371],
        [111.93244545164555, -7.112176997696622],
        [111.84013944486253, -7.112176997696622]
    ]]
}

# Ambil data NO₂ Sentinel-5P
s5post = connection.load_collection(
    "SENTINEL_5P_L2",
    temporal_extent=["2025-06-01", "2025-10-19"],
    spatial_extent={"west": 111.84013944486253, "south": -7.201964283327371,
                    "east": 111.93244545164555, "north": -7.112176997696622},
    bands=["NO2"]
)

# Masking nilai negatif
def mask_invalid(x):
    return x < 0

s5p_masked = s5post.mask(s5post.apply(mask_invalid))

# Agregasi harian dan spasial
daily_mean = s5p_masked.aggregate_temporal_period(period="day", reducer="mean")
daily_mean_aoi = daily_mean.aggregate_spatial(geometries=aoi_geojson, reducer="mean")

# Eksekusi batch dan unduh hasil dalam format CSV
job = daily_mean_aoi.execute_batch(out_format="CSV")
results = job.get_results()
results.download_files("no2_bojonegoro_dataset")

# Baca hasil
df = pd.read_csv("no2_bojonegoro_dataset/timeseries.csv")
df["date"] = pd.to_datetime(df["date"])
df["month"] = df["date"].dt.to_period("M")

# Hitung rata-rata bulanan
df_monthly = df.groupby("month", as_index=False)["NO2"].mean()

# Visualisasi tren bulanan
plt.figure(figsize=(10,5))
plt.plot(df_monthly["month"].astype(str), df_monthly["NO2"], marker="o", color="blue")
plt.title("Rata-rata Bulanan NO₂ di Bojonegoro (Sentinel-5P)")
plt.xlabel("Bulan")
plt.ylabel("Konsentrasi NO₂")
plt.grid(True)
plt.xticks(rotation=45)
plt.show()

Tahap Pra-pemrosesan Data Menangani Nilai Hilang#

Sebanyak 36 nilai hilang ditemukan dalam data. Untuk mengatasinya digunakan metode interpolasi linear, diikuti dengan forward-fill dan backward-fill agar tidak ada celah data kosong.

Preprocessing bertujuan menjadikan data mentah siap dipakai model, mengurangi noise, dan memastikan kontinuitas seri waktu.

Deteksi missing: setelah unduh, hitung jumlah hari tanpa nilai (missing). Penyebab: pengukuran tertutup awan, threshold QA yang rendah, atau kegagalan pengolahan.

Masking nilai negatif: nilai NO₂ negatif pada produk satelit biasanya menandakan noise atau artefak. Menghapus atau memask-nilai ini menghindarkan model belajar dari nilai tidak fisik.

Interpolasi linear: mengisi missing menggunakan garis lurus antar titik tetangga cocok bila data berubah relatif halus antar hari. Interpolasi tidak cocok bila ada lonjakan ekstrem (mis. kebakaran) — bisa meratakan peristiwa penting.

FFill / BFill: setelah interpolasi, gunakan forward/backward fill untuk kasus missing di awal atau akhir seri (ketika interpolasi tidak bisa karena tidak ada neighbor). Namun isi yang di-ffill di awal bisa berpotensi bias jika periode awal berbeda pola musiman.

Normalisasi / scaling: penting sebelum modeling KNN karena KNN sensitif pada skala fitur. Di sini fitur adalah lag NO₂ — StandardScaler (mean 0, var 1) umum dipakai. Jika dataset mengandung outlier besar, pertimbangkan RobustScaler.

Pengecekan distribusi: lihat histogram dan boxplot NO₂; jika distribusi sangat miring, transformasi (log atau Box-Cox) mungkin membantu stabilkan varians dan meningkatkan performa prediktif.

Catatan: dokumentasikan setiap transformasi (mengapa dan bagaimana) agar hasil bisa direproduksi.

Pembentukan Data Supervised Konsep#

Data deret waktu diubah menjadi bentuk supervised learning, di mana nilai masa depan diprediksi berdasarkan sejumlah nilai sebelumnya (lag). Semakin banyak lag digunakan, semakin banyak konteks historis yang diperhitungkan.

Mengonversi seri waktu menjadi dataset supervised berarti membuat fitur dari nilai historis (lag) dan target berupa nilai masa depan.

Konsep lag: lag_1 = nilai NO₂ satu hari sebelum target; lag_5 = 5 hari sebelum, dsb. Model akan mempelajari relasi antara lag-lag ini dan target.

Pemilihan lag: tidak ada aturan baku; gunakan pengetahuan domain (mis. karakteristik atmosfer lokal, hari kerja vs weekend) atau metode data-driven (autocorrelation function / partial ACF) untuk menentukan lag yang paling informatif. Biasanya periksa ACF/PACF untuk memilih cutoff.

Jumlah sampel efektif: semakin besar lag, semakin banyak baris hilang di awal (karena butuh lebih banyak history). Jadi lag besar mengurangi data latih dan dapat menurunkan generalisasi.

Overfitting vs Underfitting: lag terlalu kecil bisa membuat model underfit (tidak menangkap tren jangka pendek); lag terlalu besar bisa memasukkan noise/historis yang tidak relevan, meningkatkan varians model.

Shuffling & split: untuk time series jangan shuffling acak tanpa hati-hati. Meski contoh memakai train_test_split acak, praktik terbaik time-series adalah split berdasarkan waktu (mis. train sampai t, test setelah t) untuk menghindari kebocoran data masa depan. Karena KNN tidak memerlukan urutan, tapi target berasal dari waktu berikutnya — jadi gunakan split temporal untuk evaluasi yang realistis.

Contoh tambahan: periksa korelasi antar lag (multicollinearity). Jika lag saling berkorelasi tinggi, beberapa model bisa terdampak; KNN kurang sensitif namun tetap perlu diperhatikan.

Langkah Umum

Tentukan jumlah lag yang akan digunakan (misalnya 3 atau 5).

Geser data ke belakang sebanyak jumlah lag tersebut untuk membuat kolom baru seperti lag_1, lag_2, dan seterusnya.

Nilai aktual setelah pergeseran dijadikan sebagai target atau nilai keluaran model.

Catatan

Semakin besar jumlah lag yang dipilih, semakin banyak informasi historis yang dimiliki model. Namun, terlalu banyak lag dapat menyebabkan model menangkap noise atau pola yang tidak relevan. Karena itu, pemilihan jumlah lag harus dilakukan secara hati-hati agar model tetap sederhana namun tetap memberikan hasil yang akurat.

# ==========================
# Tahap 3: Data Supervised
# ==========================
def create_columns(lag=5):
    columns = [f"lag_{i}" for i in range(lag, 0, -1)]
    columns.append("target")
    return columns

def create_supervised_data(data, lag=5):
    data_supervised = []
    for i in range(lag, len(data)):
        row = data[i-lag:i+1]
        data_supervised.append(row)
    return pd.DataFrame(data_supervised, columns=create_columns(lag))

# Membuat data untuk beberapa lag
data_clean = pd.read_csv("no2_bojonegoro_dataset/clean_data.csv")
no2_values = data_clean["NO2"].to_list()

for lag in [1, 2, 3, 4, 5, 10]:
    supervised_df = create_supervised_data(no2_values, lag)
    supervised_df.to_csv(f"no2_bojonegoro_dataset/supervised_lag_{lag}.csv", index=False)

Modeling — Menggunakan KNN Regression#

Penjelasan Singkat

K-Nearest Neighbors (KNN) Regression adalah pendekatan non-parametrik untuk regresi yang membuat prediksi berdasarkan kemiripan titik data. Untuk memprediksi nilai pada suatu titik, KNN mencari sejumlah k tetangga terdekat dalam ruang fitur dan menggunakan nilai output tetangga tersebut (mis. rata-rata) sebagai prediksi. Tidak seperti model yang memaksakan bentuk fungsi tertentu (mis. linear), KNN bergantung pada struktur lokal data: ia mengasumsikan bahwa contoh yang berdekatan secara fitur cenderung memiliki nilai target yang serupa.

Catatan Praktis

KNN tidak menghasilkan persamaan eksplisit — interpretasinya datang dari pengamatan tetangga terdekat.

Karena KNN memakai ukuran jarak, skala fitur harus disejajarkan (oleh sebab itu dilakukan normalisasi dengan StandardScaler).

Pemilihan jumlah tetangga (n_neighbors) mempengaruhi bias-variance: k kecil meningkatkan varians (lebih sensitif pada noise), sedangkan k besar cenderung meratakan prediksi.

Pada contoh ini, model diuji pada beberapa set data yang dibuat berdasar lag berbeda (lag_1 sampai lag_5, serta lag_10) untuk membandingkan performa antar konfigurasi input historis.

KNN Regression memprediksi target dengan rata-rata nilai target dari k-tetangga terdekat di ruang fitur.

Parameter utama: n_neighbors (k), metric jarak (default Euclidean), dan weighting (uniform atau distance). k kecil → model lebih fleksibel tapi rawan noise; k besar → lebih stabil namun bisa oversmooth.

Scaling penting: karena KNN menggunakan jarak, setiap fitur harus berada pada skala yang serupa; itulah mengapa StandardScaler dipakai.

Cross-validation: tuning k dan opsi weighting idealnya dengan cross-validation. Untuk time series gunakan teknik CV bertingkat waktu (TimeSeriesSplit) bukan CV acak.

Interpretasi hasil: KNN tidak memberikan koefisien atau fungsi eksplisit — interpretabilitas melalui analisis tetangga (mis. melihat contoh tetangga untuk sebuah prediksi).

Kelemahan KNN: sensitif terhadap noise, membutuhkan memori (menyimpan dataset latih), dan keputusan jarak rentan terhadap fitur tak relevan. Namun gampang diimplementasikan dan kadang sangat efektif di data berstruktur lokal seperti lag-based time series.

Alternatif: model linier, Random Forest, XGBoost, atau model time-series khusus (ARIMA, SARIMA, Prophet, LSTM) jika ingin perbandingan performa.

Praktik: jalankan grid search untuk k (mis. 1–20), coba weighting distance, dan gunakan TimeSeriesSplit untuk validasi.

def create_columns(lag=5):
    columns = []
    for i in range(lag, lag - lag, -1):
        columns.append("lag_" + str(i))
    columns.append("target")
    return columns

def create_supervised_data(data, lag=5):
    data_supervised = []
    for i in range(lag, len(data)):
        row = data[i-lag:i+1]
        data_supervised.append(row)
        columns = create_columns(lag)

    supervised_df = pd.DataFrame(data_supervised, columns=columns)
    return supervised_df
